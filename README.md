## 1.プロジェクトの概要

###  プロジェクト名

競技プログラミング(AtCoder)のコード管理プロジェクト

### プロジェクトの目的

競技プログラミングの勉強やコンテストで作成したプログラムを保管する。


## 2.主要技術

| 言語・フレームワーク | バージョン |
| -------------------- | ---------- |
| C++ (GCC)            | 20（12.2） |


## 3.コマンド一覧

| コマンド                                               　| 実行する処理                         |
| ------------------------------------------------------ | ------------------------------------|
| bd <ビルド対象のcppファイルのパス>                       | 対象ファイルのコンパイル               |
| io                                                     | in.txtを元に./a.outを実行し、結果をout.txtに格納する  |
| io term                                                | in.txtを元に./a.outを実行し、結果をターミナルに出力する  |

## 4.ディレクトリ構成

```
.
├── sh/                        # シェルスクリプト置き場
│   ├── build.sh              # ソースコードのビルド用スクリプト（g++など）
│   └── io.sh                 # 入出力テストを自動化するスクリプト

├── ac-library/               # AtCoder公式の競技用ライブラリ（ACL）

├── src/                      # 自作ソースコード（問題ごとのファイルを配置）

├── .gitignore                # Gitで追跡しないファイル・フォルダの設定
├── in.txt                    # テスト用の標準入力データ
├── out.txt                   # テスト結果出力ファイル
├── a.out                     # コンパイル後の実行ファイル（build.shで生成）
└── README.md                 # このプロジェクトの説明ファイル

```
